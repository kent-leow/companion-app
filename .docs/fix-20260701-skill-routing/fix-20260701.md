# Fix: Skill Routing Not Triggering for Real-Time Queries

## Issues

1. [x] `main.rs` bypasses orchestrator — calls `chat_stream` directly, never routes through planner/agents
2. [x] Planner prompt has no awareness of available skills (web-search)
3. [x] `is_simple_query` wrongly classifies real-time questions as "simple" (word count ≤10)
4. [x] Agent pool has no mechanism to execute skills — sub-agents are LLM-only

## Root Cause

The orchestrator pattern exists in code but is **never invoked** from the main loop. The app is currently a thin streaming chat wrapper with no agentic behavior.

## Fix Plan

1. Wire `Orchestrator` into `main.rs` — route queries through it before falling back to direct stream
2. Pass skill registry to planner so it can dispatch `web-search` tasks
3. Fix `is_simple_query` to detect time-sensitive/real-time patterns
4. Enable agent pool to invoke skill executor when a sub-agent task maps to a skill
